## Observations (Non-Blocking)

A few items that aren't findings but are worth noting:

The async refactor was applied consistently across the three documents. The `recommend_architecture` tool name has been fully eliminated (zero occurrences across all three docs). The narrative walkthrough in the architecture doc reads naturally with the async model. The prompt-file orchestration pattern is well-explained. The developer guide's §2.5 rewrite is genuinely good — the Chat-pane progress example sets the right expectations.

The AlloyDB Task Store is a pragmatic choice for task state — it's managed, auto-scales, and has native TTL support. The 24-hour TTL is generous enough for timezone differences without becoming a storage cost concern.

The decision to keep `validate_composition` and `assemble_blueprint` as synchronous tools (fast, deterministic, no background pipeline needed) is correct and avoids unnecessary complexity.

The MCP protocol version bump to 2025-11-25 (from 2025-03-26) is accurate — Tasks was introduced in that revision. The compatibility table correctly notes that most coding agents handle Tasks via prompt-file polling rather than native client support.
